DJI Cloud API - Aircraft Properties (Pilot-to-Cloud MQTT)
Source: https://developer.dji.com/doc/cloud-api-tutorial/en/api-reference/pilot-to-cloud/mqtt/others/aircraft/properties.html
Extracted from MHTML archive.

This document lists the thing model properties for aircraft devices in the
DJI Cloud API pilot-to-cloud MQTT integration. Properties are reported via
thing/product/{sn}/osd (pushMode 0) or thing/product/{sn}/state (pushMode 1),
and writable properties can be set via thing/product/{sn}/property/set.

================================================================================

### Device Property List

-
pushMode:

 - 0: Stable frequency data. Device will report in the frequency of 0.5HZ. (Topic:thing/product/{sn}/osd)
 - 1: State data. It is reported when the state changes. (Topic:thing/product/{sn}/state)

-
accessMode:

 - r: Property can only be read
 - rw: Property can be read and written (Topic:thing/product/{sn}/property/set)
