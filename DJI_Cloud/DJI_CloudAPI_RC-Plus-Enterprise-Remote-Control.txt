# DRC-heart beat

The seq at the same level as data is an incrementing sequence number that ensures the sequential execution of commands.

Topic: thing/product/{gateway_sn}/drc/down

Direction: down

Method: heart_beat

Data:

Column

Name

Type

Constraint

Description

timestamp

Heartbeat transmission timestamp

int

{"max":"","min":"","step":"","unit_name":"milliseconds / ms"}

The heartbeat protocol allows the users to determine if the DRC (Data Relay Channel) link is active. If no heartbeat response is received within a certain period, error may occur to the DRC link, and a retry can be initiated. If no heartbeat protocol is received for over 1 minute, the device will consider the DRC link idle and exit the DRC link.

Example:

{
"data": {
"timestamp": 1670415891013
},
"method": "heart_beat",
"seq": 1
}

Topic: thing/product/{gateway_sn}/drc/up

Direction: up

Method: heart_beat

Data:

Column

Name

Type

Constraint

Description

timestamp

Heartbeat transmission timestamp

int

{"max":"","min":"","step":"","unit_name":"milliseconds / ms"}

The timestamp when the heartbeat was sent, not the device's time, to help the cloud calculate the delay based on the send-receive interval.

Example:

{
"data": {
"timestamp": 1670415891013
},
"method": "heart_beat",
"seq": 1
}
