# Device Property Push
Topic: thing/product/ {device_sn} /state State Data: Device reports when the state changes
Topic: thing/product/ {device_sn} /osd Fixed Frequency Data: Device will report at a frequency of 0.5 Hz
Direction: up
Notes on API
Payload property push refers to the push of properties of the payload mounted on the aircraft, such as the camera. A payload is uniquely identified by a payload index, formatted as {type-subtype-gimbalIndex} (product type-subtype-mount position). Refer to Product Supported open in new window for the values of type, subtype, and gimbalIndex.
The protocols currently involving the push of payload properties include the device property push of gateway devices, and the livestream capability update of the livestream function. For device property push , the content of payload property pushing includes information about the camera itself, such as the angles of gimbal tilt, pan, and roll. For livestream capability update , the content of payload property pushing is more about the camera's capabilities during livestream, such as the maximum number of streams that can be pushed simultaneously. Refer to the example below for the specific structure of payload properties. It should be noted that the device properties of the remote controller are pushed in a single message, while the device property pushing of the dock is done in multiple pushes.

data Content text {} Refer to the device's property for content
» camera_index type_subtype_gimbalindex text {} gimbalIndex refers to the camera position. Refer to Product Supported open in new window for the values of type, subtype, and gimbalIndex.
