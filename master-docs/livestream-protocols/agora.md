# Agora livestream transport

**Agora** — third-party real-time engagement platform whose "Interactive Live Streaming Standard" is one of the transport options DJI devices expose via `live_start_push`. Device joins an Agora channel as a broadcaster and publishes its media stream via the Agora RTC SDK; viewers subscribe via Agora SDKs or the Agora Cloud Recording / Agora On-Premise deployment.

See [`README.md`](README.md) for the device-support matrix across all four protocols and scope rules. MQTT signaling to start / stop / qualify a stream lives in [`../mqtt/dock-to-cloud/services/live_start_push.md`](../mqtt/dock-to-cloud/services/live_start_push.md).

---

## 1. MQTT selector

- `url_type` = **`0`** in the MQTT `live_start_push` enums for Dock 2, RC Plus 2 Enterprise, and RC Pro Enterprise.
- **Not supported** on Dock 3 (v1.15 Dock 3 enum is `{1: RTMP, 3: GB28181, 4: WebRTC}` only).

---

## 2. `url` shape

DJI source description (verbatim, v1.15 Dock 2 / RC): `channel&sn&token&uid`.

DJI source example (verbatim, v1.15 Dock 2):

```
channel=1ZNDH360010162_39-0-7&sn=1ZNDH360010162&token=006dca67721582a48768ec4d817b7b25a86IAB4cw2JgN6iX8BpTPdc3e4S1Iendz94IFJ56aSXKvzAJei27MqF2zyCIgCLIIoBt41+YAQAAQC3jX5gAgC3jX5gAwC3jX5gBAC3jX5g&uid=50000
```

Shape: URL-encoded `key=value` pairs joined by `&`. **Not** a standard URI; the `url` field contains the raw query string alone (no `?`, no scheme). Same shape as GB28181 but different keys.

| Key | Meaning | Agora term |
|---|---|---|
| `channel` | Agora channel name to join. Typically derived from `{sn}_{camera_index}` so each stream lives on a distinct channel. | Channel Name |
| `sn` | DJI device serial number. Duplicate of the SN prefix inside `channel`; also used by DJI to key cloud-side tracking. | — (DJI-specific) |
| `token` | Agora channel-join token. Generated by the cloud using the Agora **App ID** + **App Certificate** (kept server-side; never sent to the device directly). | Dynamic Token |
| `uid` | Agora user ID the device uses as its publisher identity. `0` means Agora auto-assigns; DJI's example uses `50000`. | UID |

### 2.1 Token URL-encoding caveat

DJI explicitly warns in every Agora-enabled source file:

> *"Agora-generated tokens may contain special characters like '+', which need to be URL-encoded once. Otherwise, the Pilot side may encounter parsing errors."*

Agora RTC tokens (format starting `006…`) can include `+`, `/`, and `=` characters. DJI requires the cloud to URL-encode the token **once** — and only once — before putting it in the `url` field. In the v1.15 Dock 2 example, the token `…IgCLIIoBt41+YAQ…` contains a raw `+`; the Dock 3 / `url_type: 0` example elsewhere shows the same token after URL-encoding (`…IgA82pQE8cCoYAQ…` — a different token; the encoding convention means `%2B` for `+`, `%2F` for `/`, `%3D` for `=`). The cloud must ensure the encoding is applied exactly once to avoid double-encoding (`%252B`) or none.

### 2.2 Channel-join flow (external to DJI)

After the device receives `live_start_push`:

1. Device parses out `channel`, `token`, `uid`.
2. Device calls Agora SDK `joinChannel(token, channel, null, uid)` — the Agora SDK is embedded in DJI's firmware.
3. Device begins publishing video (and optionally audio) to the channel as a broadcaster.
4. Cloud (or any authorized Agora SDK client) joins the same channel as an audience and subscribes to `uid` = `50000`.
5. Media travels over Agora's WebRTC-derived proprietary transport — direct to any Agora SDK-backed viewer, or into Agora Cloud Recording for storage / relay.

The cloud does **not** handle the media stream directly unless it enrolls as an Agora subscriber or hosts Agora's on-premise edge nodes.

**Dock 3 dropped Agora** between v1.11 (Dock 2 / RC Pro) and v1.15 (Dock 3). The v1.15 Dock 3 `live_start_push` example payload still uses `url_type: 0` with an Agora-shaped URL — this is a copy-paste defect from the Dock 2 source, not evidence of Dock 3 Agora support. DJI's own DJI FlightHub platform ("DJI public cloud platform") is described by DJI (v1.11 prose) as running on Agora's Interactive Live Streaming Standard — the Agora path is the default / historical DJI livestream option, not a third-party integration.

## 4. Cloud-side responsibility

The cloud must have an **Agora application** (App ID + App Certificate) and be able to:

1. **Mint tokens** — generate per-channel, per-uid RTC tokens using the Agora token-generator libraries. Each token is short-lived (typical: 24 hours) and bound to a specific channel / uid. The cloud must keep the App Certificate server-side; never send it to the device or browser.
2. **Subscribe as a viewer** — any Agora-SDK client can subscribe to the channel to receive the stream. For browser playback, the cloud can run:
   - An Agora Web SDK client in a server-side node to relay / transcode.
   - **Agora Cloud Recording** — Agora's hosted recording service that dumps the channel to cloud storage.
   - **Agora On-Premise** — a deployable edge cluster that terminates the Agora connection at the cloud's perimeter.

### 4.1 Agora hybrid-cloud note

DJI's v1.11 feature-set prose describes Agora as supporting "hybrid cloud deployment" — *"The data is in the customer's private server, and then a link is opened through the air gap to the Agora's public cloud. This link channel is mainly used to upgrade and operate and maintain the privatized deployed servers."* The point is that a third-party cloud can keep video data on-premise while using Agora's public cloud only for signaling / control. Implementers worried about data residency should consult [Agora's hybrid deployment documentation](https://docs.agora.io/en/video-calling/overview/hybrid-cloud-deployment) for current options.

## 5. Worked example — complete `live_start_push` payload

MQTT message (Cloud → Device on `thing/product/{gateway_sn}/services`):

```json
{
  "bid": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxx",
  "data": {
    "url_type": 0,
    "url": "channel=1ZNDH1D0010098_39-0-7&sn=1ZNDH1D0010098&token=006dca67721582a48768ec4d817b7b25a86IADk%2Fcm%2Fdv%2BHY6qT%2FAKM6y7TcUe4lXNvZpycH7vUMAlM6pFALUKF2zyCIgA82pQE8cCoYAQAAQDxwKhgAgDxwKhgAwDxwKhgBADxwKhg&uid=50000",
    "video_id": "1ZNDH1D0010098/39-0-7/normal-0",
    "video_quality": 0
  },
  "tid": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxx",
  "timestamp": 1654070968655,
  "method": "live_start_push"
}
```

Note the URL-encoded `%2F`, `%2B` sequences in the token per §2.1. The device parses the four kv pairs, joins channel `1ZNDH1D0010098_39-0-7` as uid `50000` using the supplied token, and publishes video.
